Dream → Real Parallel WebWorkers

The Dream program in ../webworker.dream spawns WebWorkers that each run a fresh instance of the same module in a real browser Worker (separate memory), and exchanges typed string messages. First build the module: cargo run -- sample/webworker.dream (writes sample/webworker.wasm + sample/webworker.abi.json). Then serve the repository root over HTTP (e.g. npx serve .) and open /sample/interop/webworker.html. All the Worker/postMessage glue lives in runtime/dream.js.
